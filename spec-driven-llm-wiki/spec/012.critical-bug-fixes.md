## Logic Changes

### Bug 1: Missing await in runSelfImprovePropose

`diagnoseFailures` and `buildHarnessPatch` are async but called without await.

```javascript
// BEFORE (broken):
const diagnosis = diagnoseFailures(failureSummary, recentPatches); // Promise
const patch = buildHarnessPatch(diagnosis); // receives Promise

// AFTER (fixed):
const diagnosis = await diagnoseFailures(failureSummary, recentPatches);
const patch = await buildHarnessPatch(diagnosis);
```

### Bug 2: validateProfile return value misinterpreted

`validateProfile` returns `true` on success. Sandbox checks `.valid`.

```javascript
// BEFORE (always invalid):
const validation = validateProfile(candidateActive, 'candidate harness');
if (!validation.valid) { /* always enters */ }

// AFTER (fixed):
const validation = validateProfile(candidateActive, 'candidate harness');
if (validation !== true) { /* only enters on actual failure */ }
```

### Bug 3: chatCompletion called with empty config

`diagnoseFailures`, `buildHarnessPatch`, `criticEvaluate` call `chatCompletion(root, {}, ...)`.

```javascript
// BEFORE (guaranteed throw):
await chatCompletion(root, {}, messages, tools, signal);

// AFTER (fixed):
const config = await loadConfig(root);
await chatCompletion(root, config, messages, tools, signal);
```

### Bug 4: Command injection via execSync

`callMmxProposer` builds shell command with unescaped prompt.

```javascript
// BEFORE (vulnerable):
const cmd = `npx mmx text chat --message "user:${prompt}" ...`;
execSync(cmd, ...);

// AFTER (fixed):
const { spawnSync } = require('node:child_process');
const result = spawnSync('npx', ['mmx', 'text', 'chat', '--message', `user:${prompt}`, ...], { encoding: 'utf8', timeout });
```

### Bug 5: rollbackToBackup wrong filename

Looks for `.bak`, backups are `.bak.0`.

```javascript
// BEFORE (never finds):
const bakPath = statePath(root, OVERLAY_PROFILE + '.bak');

// AFTER (fixed):
const bakPath = statePath(root, OVERLAY_PROFILE + '.bak.0');
```

### Bug 6: Unhandled async errors in HTTP server

Daemon HTTP route handlers are async but not wrapped in try/catch.

```javascript
// BEFORE (crashes on error):
res.writeHead(200);
const state = await readDaemonState(root);
res.end(JSON.stringify(state));

// AFTER (survives errors):
try {
  const state = await readDaemonState(root);
  res.writeHead(200);
  res.end(JSON.stringify(state));
} catch (err) {
  res.writeHead(500);
  res.end(JSON.stringify({ error: err.message }));
}
```
