# Spec 015: Architecture Refactor — Module Boundaries

## Task/Prompt

Break down oversized modules into focused, single-responsibility modules. Resolve circular dependencies. Remove CLI logic from library code.

## Tujuan

- `agent.js` substantially smaller and focused on agent execution (`479` lines after extraction)
- `state.js` becomes a small re-export barrel (`15` lines after extraction)
- No `agent.js` ↔ `orchestrator.js` runtime circular dependency for swarm workers
- No slash command implementation in `agent.js`
- No HTTP route implementation in `daemon.js`

## Codebase Overview

- `src/agent.js` owns tool schemas, prompt construction, tool execution, autonomous-mode flow, and trace recording.
- `src/commands/chat-commands.js` owns interactive chat REPL and slash command handlers.
- `src/safety/tool-safety.js` owns permission-mode checks, git-reversible write detection, and AI safety review.
- `src/state.js` is a barrel that re-exports focused state modules under `src/state/`.
- `src/daemon.js` owns daemon lifecycle and loop scheduling.
- `src/daemon-api.js` owns HTTP API route handling.
- `src/orchestrator.js` owns swarm planning, worker/critic orchestration, and result merging.

## Logic Changes

- Slash commands are dispatched outside `agent.js`, while `startChat` receives `runAgentTask` through dependency injection.
- Tool safety decisions are delegated to a safety module; `agent.js` calls `ensureAllowed` before executing tools.
- Persistent state is grouped by responsibility: common helpers, profile state, audit logs, candidates/optimizer/status, daemon state.
- Daemon HTTP routes receive a controller object so `/stop` can request shutdown without owning daemon lifecycle internals.
- Swarm worker execution receives `runAgentTask` and `TOOL_SCHEMAS` through options instead of dynamically importing `agent.js`.

## Changes

### 1. Extract Chat Commands from agent.js

New: `src/commands/chat-commands.js`

Exports: `handleSlashCommand`, `handleConnectCommand`, `handleModelsCommand`, `handlePermissionsCommand`, `handleSelfImproveCommand`, `handleSwarmCommand`, `handleMCPCommand`, `handleSkillsCommand`

Result: `src/agent.js` imports `handleSlashCommand` and wraps `startChat(root, options)` around the command module's injected chat loop.

### 2. Extract Tool Safety from agent.js

New: `src/safety/tool-safety.js`

Exports: `TOOL_POLICY_KEYS`, `askToolPermission`, `fileTargetForTool`, `isGitReversibleFileAction`, `reviewToolSafety`, `ensureAllowed`

### 3. Break state.js

New: `src/state/profile-state.js` — profiles, overlays, backups
New: `src/state/audit-log.js` — events, traces, patches JSONL
New: `src/state/candidate-state.js` — candidate IDs, harness, scores
New: `src/state/daemon-state.js` — daemon PID, state
New: `src/state/common.js` — constants, path helpers, JSON helpers

`src/state.js` becomes re-export barrel.

Result: fixed a latent missing `applyJsonPatch` import by importing it in `profile-state.js`.

### 4. Break daemon.js

New: `src/daemon-api.js` — HTTP route handlers

Result: `daemon.js` starts the API server via `createApiServer(root, port, controller)` and keeps lifecycle/shutdown logic local.

### 5. Resolve agent.js ↔ orchestrator.js circular dep

Inject agent runner and base tool schemas as parameters to orchestrator instead of dynamic require.

Result: `runFeatureAgent` requires `options.runAgentTask`; `agent.js` and CLI swarm calls pass `runAgentTask` + `TOOL_SCHEMAS` explicitly.

## Pseudocode

```text
startChat(root, options):
  create readline interface
  for each prompt:
    if prompt starts with slash:
      handleSlashCommand(root, prompt, rl)
    else:
      runAgentTask(root, prompt, injected options)

executeTool(...):
  args = parseToolArgs(toolCall)
  ensureAllowed(root, profile, config, name, args, options)
  dispatch built-in or injected handler

runSwarm(root, prompt, options):
  plan features
  for feature batch:
    runFeatureAgent(root, feature, options with runAgentTask/toolSchemas)
  merge settled results

createApiServer(root, port, controller):
  route /status, /candidates, /trigger, /stop
  call controller.stop + controller.gracefulShutdown for /stop
```
